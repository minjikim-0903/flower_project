# 비즈니스 모델

## 서비스 개요

**꽃시장**은 꽃 도소매 업자(판매자)와 구매자를 연결하는 모바일 마켓플레이스 앱입니다.

판매자는 본인의 지점을 등록하고 상품과 가격을 관리하며, 구매자는 판매자가 올린 상품을 주문합니다. 주문이 들어오면 판매자가 알림을 받고 직접 배송(또는 픽업)을 처리합니다.

## 주요 참여자

| 구분 | 설명 |
|---|---|
| 판매자 | 꽃 도소매 업자. 지점 등록 후 상품/가격 관리 및 주문 처리 |
| 구매자 | 꽃가게 운영자, 플로리스트, 웨딩플로리스트, 일반 소비자 등 |
| 관리자 | 판매자·구매자 계정 및 플랫폼 전반 관리 |

## 핵심 흐름

```
판매자 지점 등록 → 상품/가격 등록
        ↓
구매자 상품 탐색 → 장바구니 → 주문
        ↓
판매자 주문 알림 수신 → 주문 확인 → 배송 처리
        ↓
구매자 주문 상태 확인
```

## 상품 카테고리

### 생화
장미, 백합, 튤립, 국화, 카네이션, 해바라기, 난초, 혼합, 기타

### 나무
유실수, 관상수, 침엽수, 관목, 실내식물, 대나무, 혼합, 기타

## 가격 체계

- **소매가**: 소량 구매 구매자 대상
- **도매가**: 업체 구매자(꽃가게, 플로리스트 등) 대상

---

## 수익 구조

### 기본 개념

구매자가 결제한 금액에서 PG사 수수료와 플랫폼 수수료를 차감한 후, 나머지를 판매자에게 정산하는 구조입니다.

```
구매자 결제 금액 (VAT 포함가)
        ↓
   PG 수수료 절반  (- X/2%)  ← 구매자 부담 (결제 금액에 포함)
        ↓
 플랫폼 수수료 차감  (- Y%)   ← 꽃시장 수익
   PG 수수료 절반  (- X/2%)  ← 판매자 부담 (정산 시 차감)
        ↓
  판매자 정산 금액  (나머지)
```

### 구성 요소

| 항목 | 부담 주체 | 비율 |
|---|---|---|
| PG 수수료 (구매자 분) | 구매자 | X/2% — 추후 확정 |
| PG 수수료 (판매자 분) | 판매자 | X/2% — 추후 확정 |
| 플랫폼 수수료 | 판매자 (정산 시 차감) | Y% — 추후 확정 |
| 판매자 정산 | 판매자 수령 | 나머지 |

### VAT

- 판매가는 **VAT 포함가**로 표시 및 결제
- 판매자가 VAT 신고/납부 의무 부담
- 세금계산서 발행 방식은 추후 기획 필요

> 정확한 수수료 비율 및 정산 주기, 정산 방식은 추후 기획 필요

### 고려사항

**PG사 선정**
- 토스페이먼츠, 포트원(아임포트), KG이니시스 등 비교 필요
- 카드 수수료: 일반적으로 2.2~3.3% 수준
- 계좌이체 수수료: 일반적으로 1% 수준
- 소규모 스타트업 우대 요금제 여부 확인

**플랫폼 수수료 설계**
- 소매/도매 거래별 수수료 차등 적용 여부
- 판매자 등급제 도입 시 수수료 차등 적용 가능
- 초기 런칭 시 수수료 무료 또는 낮은 수수료로 판매자 유치 전략 고려

**정산**
- 정산 주기: 주 1회 / 월 2회 / 월 1회 중 선택
- 정산 기준일: 주문 완료 기준 vs 배송 완료 기준
- 부분 취소/환불 발생 시 정산 처리 방식
- 세금계산서 발행 주체 및 시점

**법적 검토**
- 통신판매중개업 신고 필요 여부
- 전자금융거래법 적용 범위 확인
- 소비자 보호 및 환불 정책 수립

**이용약관**
- 서비스 이용약관 (구매자용 / 판매자용 분리 또는 통합)
- 개인정보 처리방침
- 위치정보 이용약관 (주소 기반 기능 사용 시)
- 마케팅 정보 수신 동의

**운영 정책**
- 판매자 입점 정책 (심사 기준, 제재 기준)
- 상품 등록 정책 (금지 품목, 사진 기준 등)
- 취소/환불 정책 (구매자 요청 vs 판매자 귀책 구분)
- 분쟁 처리 절차 (구매자-판매자 간 분쟁 시 플랫폼 개입 기준)
- 판매자 제재 정책 (경고, 판매 중지, 강제 탈퇴 기준)
