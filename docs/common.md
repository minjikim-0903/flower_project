# 공통 — 인증 시스템

## 계정 구조

하나의 계정으로 **판매자**와 **구매자** 역할을 모두 수행할 수 있습니다.
앱 내에서 역할을 전환하며 사용할 수 있으며, 별도 계정을 만들 필요가 없습니다.

```
하나의 계정
├── 구매자 모드  (기본 — 로그인만 하면 사용 가능)
└── 판매자 모드  (사업자 등록 후 활성화)
```

## 회원가입

- 이메일 + 비밀번호로 가입
- 가입 시 기본 정보 입력: 이름, 연락처
- 가입 후 기본값은 **구매자** 상태

## 로그인

- 이메일 + 비밀번호 로그인
- Supabase Auth 기반

## 판매자 전환

- 판매자로 전환하려면 **사업자등록증** 보유 필요
- 사업자등록증 인증 기능: 추후 도입 예정 (현재 보류)
- 현재는 판매자 전환 신청 시 수동 확인 또는 자가 신고 방식

## 역할 전환

- 앱 내에서 판매자 ↔ 구매자 모드 자유롭게 전환 가능
- 판매자이면서 구매자로도 주문 가능
- 구매자가 판매자 등록 후 판매 시작 가능

## 인증 기술

| 항목 | 내용 |
|---|---|
| 인증 서비스 | Supabase Auth |
| 세션 관리 | Supabase 내장 세션 |
| 상태 관리 | Zustand |
