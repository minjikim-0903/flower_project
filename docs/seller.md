# 판매자

## 개요

판매자는 본인의 가게를 등록하고, 상품과 가격을 관리하며, 들어오는 주문을 처리합니다.
판매자 모드는 사업자등록증 보유자만 사용 가능합니다.

## 판매자 유형

| 유형 | 설명 |
|---|---|
| 꽃가게 운영 | 소매 중심의 꽃가게 운영자 |
| 도매소매업 | 도매 및 소매 병행 업자 |
| 기타 | 직접 입력 |

> 판매자 유형은 니즈 파악 및 가격 정책 적용을 위한 정보입니다.

## 가게 관리

### 가게 목록 (seller/store.tsx)
- 내가 등록한 가게 전체 목록 카드 형태로 표시
- 가게별 영업중 / 휴업중 상태 표시 및 토글 가능
- 카드 탭 → 가게 수정 폼 이동
- 우상단 `+ 가게 등록` 버튼 → 가게 등록 폼 이동
- 가게 없을 시 빈 화면 + `첫 가게 등록하기` 버튼 노출

### 가게 등록/수정 (seller/store-form.tsx)
- `storeId` 파라미터 있으면 수정 모드, 없으면 신규 등록 모드
- 입력 항목: 가게명*, 주소*, 사업자등록번호, 최소주문금액, 가게 소개, 대표 사진
- 저장 완료 후 목록으로 자동 복귀 (useFocusEffect로 목록 새로고침)

## 상품 관리

### 상품 목록 (seller/products.tsx)
- 생화 / 나무 / 전체 필터 탭
- 상품카드: 품명, 카테고리, 품종, 산지, 소매가/도매가, 판매중/중지 토글
- 우상단 `+ 추가` 버튼 → 상품 등록 폼 이동

### 상품 등록 (seller/product-form.tsx)
4단계 스텝 폼으로 구성

| 단계 | 항목 |
|---|---|
| 1. 기본 정보 & 상품 특성 | 상품종류, 카테고리, 품명, 품종(카테고리별 칩+직접입력), 컬러, 꽃크기, 개화시기, 신선도, 특징, 가시·향기 유무 |
| 2. 판매 조건 | 소매가, 도매가, 최소주문수량, 최소도매수량, 단위, 재고, 판매기간, 대량할인 조건 |
| 3. 배송 정보 | 출하지(산지), 배송가능지역, 배송방법, 배송소요일, 배송비, 주문마감시간, 보냉포장 여부 |
| 4. 일정 & 추천 | 출하가능요일, 입고예정요일, 수확일, 판매시즌, 추천구매자 유형, 유의사항 |

#### 품종 선택 (categories.md 기준)
| 카테고리 | 품종 |
|---|---|
| 장미 | 스탠다드 장미, 스프레이 장미 |
| 국화 | 스탠다드 국화, 스프레이 국화, 소국 |
| 백합 | 오리엔탈, 아시아틱, LA 하이브리드 |
| 카네이션 | 스탠다드 카네이션, 스프레이 카네이션 |
| 튤립 | 싱글 튤립, 더블 튤립, 프린지 튤립 |
| 해바라기 | 스탠다드 해바라기, 미니 해바라기 |
| 난초 | 호접란, 덴드로비움, 심비디움 |
| 기타 | 안개꽃, 루스커스, 유칼립투스, 거베라, 프리지아, 수국, 작약 |

## 주문 처리 흐름

```
구매자 주문 접수
      ↓
판매자 알림 수신
      ↓
주문 확인 → 처리 상태 변경 (접수 → 준비 중 → 배송 중 → 완료)
```

> 배송은 판매자가 직접 처리하며, 앱 내 배송 시스템은 제공하지 않습니다.
